# ADR-001: Использование OpenAPI 3.0 для контрактов API

- **Статус**: Принято  
- **Дата**: 24.12.2025  
- **Автор**: Валентина Погр, Middle System Analyst  

## Контекст
В рамках интеграции внешнего поставщика с legacy-системой требуется единый, версионируемый контракт.  
Ранее использовались Word-документы — возникали расхождения между спецификацией и реализацией.

## Решение
Использовать **OpenAPI 3.0** в формате YAML:
- хранится в Git,
- поддерживает `$ref`, примеры, схемы,
- позволяет генерировать Postman-коллекции и mock-серверы.

## Обоснование
- ✅ «Живая документация»: обновление → автоматическая валидация  
- ✅ Опыт автора: работа с OpenAPI в legacy-интеграциях  
- ✅ Поддержка со стороны инструментов (Redoc, Swagger UI)

## Последствия
- ✅ Согласованность между разработкой и тестированием  
- ⚠️ Требуется обучение команды (план: доклад в Q1 2026)
